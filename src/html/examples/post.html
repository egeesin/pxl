Features

Theme:

  • Mobile-first responsive layout.
    • Also looks good on ultra-wide monitors.
  • (Almost) every element has multiple designs to serve content in multiple ways.
  • Scalable and maintainable codebase written with several naming conventions and methodologies like OOCSS, BEVM in mind
    • Opinionated styles as an option.
  • No CSS frameworks used.
    • Only homemade components.
  • Improved typography with vertical rhythm and modular scale.
  • Written with modern specifications like custom properties, grid, mediaMatch…
  • Supports every web browser except every IE, Opera Mini, and other browsers that has global usage lesser than 0.33%
    • browserl.ist
  • Default font as Inter UI, open source sans-serif typeface.
    • System fonts as fallback.
  • Default color palette is Solarized.
    • You can also customize main/complementary colors in CSS variables.
  • Touch-friendly dropdown menu.

Environment:

  • Everything is automated with Gulp.js.
  • Works with Autoprefixer, PostCSS and PostHTML plugins.

Installation & Usage

Dependencies: npm, gulp, git

git clone https://github.com/egeesin/pxl && cd pxl
npm install -D

gulp build 
mv dist/* .../path/to/my/dir/

The intention is to support modern browsers to use newest tools. Supporting legacy browsers isn’t main focus yet.

At this time,

  • All IE versions, Opera Mini unsupported,
  • Edge is partially supported,
  • All other modern browsers has bigger than 0.33% usage is fully supported.

Methodology

Before we begin, it’s useful to know basic knowledge/logic of CSS methodologies/naming conventions like BEM, OOCSS, BEVM, Harry Roberts’ namespaces. Check out them if you haven’t.

In this theme, I tried to combine a few of them to improve readability. In BEM naming convention, class name is declared like this: .block__element--modifier but in my opinion I find modifier part is limited beside BEVM so I decided to go with BEVM. To improve readability, I wrote name groups with camelCase instead of separating-like-this. So you can track blocks, modifier etc. easily without having problem with naming.

I also wrote modifier and variant classes with extracting block and element names. If you familiar with UNIX command line this will probably help.

// Not like this
block block--modifier
	block__element block__element--modifier

// But this
block --modifier
	block__element --modifier
block -variant

To make modifier connected with block in CSS: .block.-modifier Note: Classes begins with a tick or an underscore, doesn’t support in IE8 or older browsers.

One last thing is namespaces that mentioned earlier. I use all except Is-Has states, JavaScript and QA namespaces.

Typography

It’s a huge part of this theme. Every width, height, padding, margin unit are em-based. Heading elements are proportionally sized by modular scale and each typographic element margin resized to fit vertical rhythm. Thanks to CSS variables, everything is reconfigurable.

Concepts

Most of them has pretty simple purposes. As far as it’s simple, all of them can be reusable, some of them can be placed recursively.

Card | c-card

First thing comes in mind when placing content, that would be a card. It’s really similar concept that mostly seen in Material Design, iOS App Store. Cards are portable and contains any type of content. It can be article container or widget box. With custom options and some JS it can be even a progressive modal.

Height of a card must be decided by content itself.

Card must not part of a body but a thing in the body. That’s the main difference between frames and cards.

Frame | c-frame

Frames are for full-sized content like jumbotron, videos. It must look part of body. It must contain, not being contained. Width of a frame is also width of viewport.

Navigation | o-nav

A navigation menu must let reader navigate through site or category. It doesn’t have be limited to be a navigation bar. There are a lot of options.

Tree View | o-nav -viewTree

Tree view is the most simple form of a navigation concept.

Dropdown View | o-nav -viewDropdown

A navigation bar with only first-level shown. Higher levels displayed by hovering cursor on its parent (in mobile, by tapping its parent). It’s useful to make complex navigations look compact.

With -collapsible option, it can be hidden and toggled by a (c-toggleTrigger) button. Highly recommended for handheld devices.

Dropdown navigation placement can also changed by -positionTop and -positionSide

Note: All functionalities except arrows works without JavaScript.
